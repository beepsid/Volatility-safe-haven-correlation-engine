Market Volatility and Safe-Haven Correlation Engine

An end-to-end financial data pipeline analyzing the relationship between Nifty 50 equity volatility and Gold ETF performance during market stress periods.

Python pandas yfinance SQL Power BI 5 Years of Data

Business Case

A core question in portfolio risk management is whether gold acts as a reliable safe haven when equity markets drop. This project quantifies that relationship using 5 years of daily market data, engineers financial features from raw prices, and surfaces the findings through an interactive Power BI dashboard.

The central hypothesis: on days when the Nifty 50 drops more than 1.5%, does gold tend to hold or gain value?

Engineered Features

Raw price data is transformed into analytical features before any modeling or visualization.

Feature Description
nifty_return Daily percentage return of Nifty 50
gold_return Daily percentage return of GOLDBEES ETF
nifty_vol_30d 30-day rolling standard deviation of Nifty returns
nifty_rsi_14 14-day Relative Strength Index
high_vol_flag True if Nifty daily return is below -1.5%
gold_on_crash_days Gold return on flagged crash days, null otherwise

Key Findings

-1.5%

Crash day threshold. Days where Nifty drops beyond this are tagged for safe-haven analysis.

5 Years

Daily historical data from 2021 to 2026, covering multiple market cycles including post-COVID recovery and rate hike periods.

Inverse

Gold shows a stronger inverse correlation with Nifty returns during high volatility periods than during normal market conditions.

SQL Layer

The database layer models the time-series data with proper indexing and two analytical views built using SQL window functions.

Moving Averages View

SELECT trade_date, nifty_price,
  AVG(nifty_price) OVER (
    ORDER BY trade_date
    ROWS BETWEEN 49 PRECEDING AND CURRENT ROW
  ) AS ma_50,
  AVG(nifty_price) OVER (
    ORDER BY trade_date
    ROWS BETWEEN 199 PRECEDING AND CURRENT ROW
  ) AS ma_200
FROM market_data;

Gold Return During Crash Weeks

SELECT
  DATE_TRUNC('week', trade_date) AS week,
  AVG(gold_return) AS avg_gold_return,
  COUNT(*) AS crash_days
FROM market_data
WHERE high_vol_flag = TRUE
GROUP BY week
ORDER BY week;

How to Run

1. Install dependencies

pip install yfinance pandas numpy

2. Run the pipeline

python script/data_pipeline.py

3. Open the dashboard

Load data/market_data.csv into Power BI via Get Data > Text/CSV, then open dashboard/PowerBI_Dashboard.pbix.

4. Optional: PostgreSQL

Run sql/create_table.sql to create the schema, then load the CSV into the table using pgAdmin or the COPY command.
